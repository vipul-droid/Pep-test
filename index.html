Theory

We are learning about functions now, but can you believe that we have been using them from the time when we made our first program. Remember a line you have been writing all along : I am talking about the public static void main(String[] args). Main() is a function here. The execution of our program starts from the main().

A function is a block of code that executes when called and that does a particular task whenever you call it. A function is always written inside a class.

  • User-Defined Functions - These are the ones we create based on our requirements.
  • Standard Library Functions - The functions are already defined in Java Libraries, you can directly use them in your programs. For example : str.length() would give you the length of string str.

We calculate n factorial by multiplying all the natural numbers from 1 to n. And we follow the same process for calculating n - r factorial. For better understanding of this code, you can watch the video [0:15 - 3:35].

But if you think about it, is it a correct practice? because we are writing the same piece of code twice. Now think about, what if you had to calculate factorial a few more times. The same ‘for loop’ would have appeared in the program multiple times. To avoid this redundancy of code, we use functions or methods.

Run the code

But you might argue that you’ll just have to copy and paste the code and you don’t have a problem with it. Now, let me tell you a scenario, suppose if it's a really long program and there is a task (for example : telling if a number is prime or not) that needs to be performed multiple times in it. So, supporting your argument, you'll copy-paste the code multiple times. Now, your teacher tells you an efficient way of finding prime numbers. Now, you’ll have to change the code everywhere in the program, whereas in the case of functions, you would only have to change it once. Also, if you accidentally made a mistake somewhere, that would be copied in the entire program and you’ll have to parse the whole code again. You can now watch the video from [3:35 - 5:25]

The input that this function receives is an integer n. A function can receive any number of inputs of the same or different data types. If a function doesn’t receive any input, then we leave those brackets empty. The inputs in Java are called parameters. We’ll read about public and static later.
